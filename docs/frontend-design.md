# AgentWeb 前端 UI 设计需求文档

## 目标
为 AgentWeb 智能体市场平台设计前端页面。风格：现代、极简、开发者友好。组件库：Tailwind + shadcn/ui。

---

## 页面清单与设计要求

### 1. 首页 / Agent 市场（核心页面）

**路径：** `/`

**布局：**
- 顶部导航栏：Logo(AgentWeb) | 搜索框 | 登录/注册 按钮（未登录）| 头像+下拉菜单（已登录）
- Hero 区域：一句话 slogan "发现并调用领域专家 Agent" + 搜索框（居中大号）
- 分类标签栏：法律 | 医疗 | 代码 | 数据 | 翻译 | 全部... （横向滚动 pill 标签）
- Agent 卡片网格（3列 responsive）：
  - 每张卡片包含：Agent 名称、描述（2行截断）、分类标签、价格（¥XX/百万tokens）、状态指示灯（绿色在线/灰色离线）、调用次数、平均响应时间
- 排序：按调用量 | 按价格 | 最新上架
- 分页器

**关键交互：**
- 点击分类标签筛选
- 搜索实时过滤
- 点击卡片进入详情页

### 2. Agent 详情页

**路径：** `/agents/:id`

**布局：**
- 面包屑：市场 > 分类 > Agent名称
- 左侧主区域（2/3宽）：
  - Agent 名称 + 状态徽标（Online/Offline）
  - 作者信息
  - 版本号
  - 完整描述
  - 能力列表（Capabilities）：每个能力显示为可展开的卡片，展开后显示输入/输出 schema
  - 调用示例（代码块，展示 curl 命令，可切换 Python/JavaScript SDK 示例）
- 右侧信息栏（1/3宽）：
  - 价格卡片：¥XX / 百万 tokens
  - 统计面板：总调用次数 | 平均响应时间 | 在线率
  - "开始使用" 按钮 → 未登录跳登录，已登录跳到 API Key 管理或直接展示调用方式
  - 分类标签
  - 创建时间 / 更新时间

### 6. 开发者控制台 - 我的 Agent

**路径：** `/console/agents`

**布局：**
- 标题 + "创建新 Agent" 按钮
- Agent 表格/卡片列表：
  - 名称 | 状态 | 版本 | 价格 | 调用次数 | 创建时间 | 操作（编辑/删除）
- 空状态：引导创建第一个 Agent

**创建 Agent 页面：** `/console/agents/new`

设计理念：Agent 自己最了解自己。开发者只需提供连接信息，平台自动拉取 Agent Card，开发者仅补充平台侧字段即可发布。

**Step 1 — 连接 Agent（极简输入）**
- Agent 端点 URL 输入框（placeholder: `https://your-agent.example.com`）
- Agent Key 选择（下拉，列出已创建的 agent_key；若无则引导先创建）
- "拉取信息" 按钮
- 点击后平台请求 `{endpoint}/.well-known/agent.json`，获取 Agent Card
- 加载状态：显示 spinner + "正在获取 Agent 信息..."
- 失败状态：提示 "无法连接到该地址，请检查 URL 和 Agent 是否在线"

**Step 2 — 确认 Agent 信息（自动填充 + 人工补充）**
- 以下字段由 Agent Card 自动填充，**开发者可编辑覆盖**：
  - Agent 名称（只读展示，旁边有"编辑"图标可切换为可编辑）
  - 描述（多行文本，预填）
  - 版本号（预填）
  - 能力列表（预填，每个能力展示为可展开卡片：名称 + 描述 + input/output schema）
- 以下字段需开发者手动填写（Agent Card 标准中无）：
  - 分类选择（下拉：法律 / 医疗 / 代码 / 数据 / 翻译 / 其他）
  - 定价（每百万 tokens 单价，数字输入，单位 ¥）

**Step 3 — 确认发布**
- 信息预览卡片（模拟市场上的展示效果）
- "发布到市场" 按钮
- 发布成功后跳转到"我的 Agent"列表，提示 "Agent 已上架，连接 WebSocket 后即可接收调用"

**编辑 Agent 页面：** `/console/agents/:id/edit`
- 同 Step 2 布局，预填现有数据
- 增加 "重新拉取" 按钮：从 Agent 端点重新获取最新 Agent Card 并刷新表单
- 保存后返回列表

## 全局设计规范

**色调：**
- 主色：蓝色系（#2563EB 或类似），代表科技/信任
- 辅色：绿色表示在线/成功，红色表示离线/错误，灰色表示禁用
- 背景：浅灰白色（#F9FAFB），卡片白色

**状态指示：**
- Agent 在线：绿色圆点 + "在线"
- Agent 离线：灰色圆点 + "离线"

**响应式：**
- 桌面优先，但市场页面需适配移动端（卡片单列）
- 控制台桌面端即可

**组件风格：**
- shadcn/ui 默认风格：圆角、微阴影、清晰的层次
- 代码块：深色背景 + 语法高亮
- 数据为空时：友好的空状态插图 + 引导文案

## 用户流程

```
未登录用户：
  首页浏览 Agent → 点击 Agent → 查看详情 → 点击"开始使用" → 跳转登录 → 登录/注册 → 回到详情页

消费方：
  登录 → 控制台 → 创建 api_key → 复制 Key → 参考详情页的调用示例 → 查看消费明细

提供方：
  登录 → 控制台 → 创建 agent_key → 创建 Agent（填入端点 URL + 选择 Key → 平台自动拉取 Agent Card → 补充定价/分类 → 发布） → Agent 通过 WebSocket 连接平台上线 → 查看收入明细
```
